# TUI для Essentia на основе Bubble Tea

## Обзор
Создание TUI (Text User Interface) для приложения Essentia с использованием библиотеки [charmbracelet/bubbletea](https://github.com/charmbracelet/bubbletea). TUI будет иметь несколько экранов: Home, Processing, Metrics, How-does-it-work.

## Phase 1: Базовая структура и навигация
**Acceptance Criteria:**
- [x] Создана базовая структура TUI приложения с bubbletea
- [x] Реализована навигация между экранами (Home, Processing, Metrics, How-does-it-work)
- [x] Добавлен шапка с названиями разделов (горизонтальное меню)
- [x] При выборе раздела он подсвечивается оранжевым цветом в стиле Claude Code
- [x] Навигация работает с клавишами Tab/Shift+Tab или стрелками
- [x] Реализовано переключение между экранами по Enter

**Files to create/modify:**
- cmd/tui/main.go - точка входа TUI приложения
- internal/tui/app.go - основная структура приложения
- internal/tui/models/home.go - модель домашнего экрана
- internal/tui/models/processing.go - модель экрана processing
- internal/tui/models/metrics.go - модель экрана метрик
- internal/tui/models/help.go - модель экрана справки
- internal/tui/styles.go - стили и цвета
- go.mod - добавление зависимостей bubbletea и lipgloss

**Expected output:** Базовая TUI приложение с навигацией между 4 экранами. При выборе раздела в шапке он подсвечивается оранжевым. Enter переключает на выбранный экран.

## Phase 3: Экран Processing
**Acceptance Criteria:**
- [ ] Отображается список задач (jobs) в pipeline
- [ ] Каждая задача показывает: ID, статус (waiting, processing, done, error)
- [ ] Справа от списка отображается детальная информация о выбранной задаче
- [ ] Детальная информация включает: id, status, stage, total, done, attempts, max-attempts
- [ ] Поддержка прокрутки списка задач (стрелки вверх/вниз)
- [ ] При выборе задачи в списке детальная информация обновляется
- [ ] Автообновление статусов (опционально, polling каждые 2 секунды)

**Files to create/modify:**
- internal/tui/models/processing.go - расширение модели экрана processing
- internal/tui/components/joblist.go - компонент списка задач
- internal/tui/components/jobdetail.go - компонент детальной информации о задаче
- internal/tui/api/jobs.go - интеграция с API для получения задач (заглушка)

**Expected output:** Экран Processing с двумя панелями: слева список задач с прокруткой, справа детальная информация о выбранной задаче. Возможность выбирать задачу и видеть её параметры.
